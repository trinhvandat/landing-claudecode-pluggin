# Ad Copy Formulas & Google Ads Constraints

Reference for Step 4 of the audience-persona skill. Use these formulas and constraints when generating ad copy.

## Google Ads Character Limits — HARD LIMITS

```
┌──────────────────────────────────────────────────────────┐
│  ELEMENT                    │  MAX CHARACTERS            │
├──────────────────────────────────────────────────────────┤
│  Headline                   │  30 characters             │
│  Description                │  90 characters             │
│  Display URL path (×2)      │  15 characters each        │
│  Sitelink title             │  25 characters             │
│  Sitelink description (×2)  │  35 characters per line    │
│  Callout extension          │  25 characters             │
│  Structured snippet value   │  25 characters             │
└──────────────────────────────────────────────────────────┘

Spaces count. Punctuation counts.
Vietnamese diacritics (ă, â, ê, ô, ơ, ư, đ) = 1 character each.
```

### Character Counting Rules

1. Count EVERY character including spaces and punctuation
2. Vietnamese example: "Trồng Răng Implant" = 18 characters (not 19)
3. Do NOT count HTML entities — only rendered text
4. If over limit: REWRITE shorter. Never truncate. Never deliver over-limit copy.

## Copywriting Frameworks

### PAS — Problem → Agitation → Solution

Best for: pain-point-heavy niches (medical, repair, urgent services)

```
HEADLINE (Problem):   Name the pain directly
                      "Răng Bị Mất? Đừng Chờ Thêm"  (26 chars)

DESCRIPTION (Agitate + Solve):
                      Amplify the consequence, then present the fix
                      "Mất răng ảnh hưởng đến sức khỏe và tự tin. Implant vĩnh viễn từ 15tr. Đặt lịch tư vấn miễn phí." (exceeded — needs rewrite)

LANDING PAGE FLOW:
  Hero      → Name the problem (PAS: P)
  Section 2 → Amplify the pain, show consequences (PAS: A)
  Section 3 → Present the solution (PAS: S)
  CTA       → Easy next step
```

### AIDA — Attention → Interest → Desire → Action

Best for: competitive markets, comparison shoppers

```
HEADLINE (Attention): Hook with something unexpected or specific
                      "Implant Từ 15 Triệu — Bảo Hành" (too long — needs split into 2 headlines)

DESCRIPTION (Interest + Desire):
                      Build interest with proof, create desire with outcomes
                      "Hơn 2000 ca thành công. Công nghệ Đức. Trả góp 0%. Đặt lịch tư vấn miễn phí hôm nay."

LANDING PAGE FLOW:
  Hero      → Attention-grabbing headline + stat
  Section 2 → Interesting details, features-as-benefits
  Section 3 → Desire: testimonials, results, transformation
  CTA       → Clear action with low friction
```

### BAB — Before → After → Bridge

Best for: transformation-focused products (cosmetic, lifestyle, results-driven)

```
HEADLINE (Before/After): Show the transformation
                         "Từ Mất Răng Đến Nụ Cười Tự Tin" (29 chars ✅)

DESCRIPTION (Bridge):
                      How we get them from before to after
                      "Chỉ 1 lần phẫu thuật. Ăn nhai tự nhiên sau 3 tháng. Đội ngũ 15 năm kinh nghiệm. Gọi ngay."

LANDING PAGE FLOW:
  Hero      → "Before" state (empathy)
  Section 2 → "After" state (aspiration)
  Section 3 → "Bridge" (how we get you there — our method/process)
  CTA       → Start the transformation
```

### 4U — Urgent, Unique, Useful, Ultra-specific

Best for: headlines only (not full frameworks)

```
Urgent:        Time pressure — "Chỉ Còn 5 Suất Ưu Đãi" (24 chars ✅)
Unique:        What only you offer — "Implant Bảo Hành Trọn Đời" (26 chars ✅)
Useful:        Clear benefit — "Ăn Nhai Tự Nhiên Ngay" (21 chars ✅)
Ultra-specific: Concrete detail — "Implant Từ 15 Triệu Đồng" (25 chars ✅)
```

## Headline Formulas (≤ 30 chars)

| Formula | Template | Example | Chars |
|---------|----------|---------|-------|
| Keyword + Benefit | "{Product} {Benefit}" | "Implant Không Đau Tại HCM" | 26 |
| Number + Promise | "{N} {Promise}" | "500+ Ca Implant Thành Công" | 27 |
| Question Hook | "{Question}?" | "Mất Răng? Giải Pháp Đây" | 24 |
| How-To | "Cách {Achieve Result}" | "Cách Có Răng Đẹp Vĩnh Viễn" | 28 |
| Social Proof | "{Proof} {Product}" | "Top 1 Nha Khoa Implant HCM" | 27 |
| Urgency | "{Time Limit} {Offer}" | "Ưu Đãi 30% — Chỉ Tháng Này" | 28 |
| Price Anchor | "{Product} Từ {Price}" | "Implant Từ 15 Triệu Đồng" | 25 |

## Description Formulas (≤ 90 chars)

| Formula | Template |
|---------|----------|
| Proof + Feature + CTA | "{Social proof}. {Key feature}. {CTA}." |
| Pain + Solution + CTA | "{Pain point}. {Our solution}. {CTA}." |
| Feature + Benefit + Urgency | "{Feature} giúp {benefit}. {Urgency}. {CTA}." |
| Trust + Offer + CTA | "{Trust signal}. {Special offer}. {CTA}." |

## CTA Copy Rules

### GOOD CTA = Action Verb + Specific Benefit
```
✅ "Đặt Lịch Tư Vấn Miễn Phí"     → action + what they get
✅ "Nhận Báo Giá Trong 24h"        → action + speed promise
✅ "Xem Kết Quả Trước & Sau"       → action + concrete content
✅ "Start Free Trial"               → action + zero-risk offer
```

### BAD CTA = Generic or Vague
```
❌ "Gửi"              → submit what? why?
❌ "Tìm Hiểu Thêm"   → learn more is a delay, not an action
❌ "Click Here"        → meaningless
❌ "Submit"            → worst possible CTA
```

### CTA for Each Persona Type
```
Price-conscious   → "Xem Bảng Giá Chi Tiết"     (emphasize transparency)
Quality-seeking   → "Đặt Lịch Với Chuyên Gia"   (emphasize expertise)
Urgent-need       → "Gọi Ngay — Hỗ Trợ 24/7"   (emphasize speed)
Comparison-shopper → "So Sánh Gói Dịch Vụ"      (emphasize information)
```

## Message Match Rules

The #1 rule of Google Ads landing pages:

```
Ad Headline  ──must match──►  Landing Page H1

IF ad says "Trồng Răng Implant Không Đau"
THEN landing page h1 MUST say "Trồng Răng Implant Không Đau"

NOT "similar". NOT "related". EXACT MATCH.
Mismatch = low Quality Score = high CPC = wasted budget.
```

When generating ad headlines in Step 4, remember: Headline 1 from each ad group becomes the landing page h1. Write it knowing it will appear in both places.

## Sitelink Best Practices

4 sitelinks per ad group. Each adds value by offering alternative entry points.

```
Pattern for service businesses:
  1. Pricing/Packages    → "Bảng Giá Dịch Vụ"
  2. Results/Reviews     → "Đánh Giá Khách Hàng"
  3. About/Team          → "Đội Ngũ Bác Sĩ"
  4. Contact/Location    → "Địa Chỉ & Liên Hệ"

Pattern for SaaS/products:
  1. Features            → "Key Features"
  2. Pricing             → "Pricing Plans"
  3. Case Studies        → "Customer Stories"
  4. Free Trial          → "Start Free Trial"
```
